# Rule Versioning

Store one YAML file per rule revision.

Naming convention:
- `ifvg_entry_v001.yaml`
- `ifvg_entry_v002.yaml`
- or date-based `ifvg_entry_2026-05-04.yaml`

Each revision should include:
1. `version`
2. narrative/setup requirements
3. execution + risk rules
4. short changelog note in `notes`
