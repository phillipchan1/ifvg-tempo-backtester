# Backtest run outputs

CSVs in this folder are **derived artifacts** — generated by `scripts/run_backtest.py` (single-TF) or `scripts/run_multi_tf.py` (multi-TF). Tracked in git so trade sets can be diffed across rule versions.

## Naming convention

Going forward:
```
runs/<rules_version>_<date_range>[_<exec_tf>][_<scope>].csv
runs/<rules_version>_<date_range>_multi_tf.csv   # multi-TF combined
```

Examples:
- `v002_2026-03_1m.csv` — v002 rules, March 2026 window, 1m execution
- `v002_2026-03_multi_tf.csv` — same window, all five TFs
- `v003_2025-Q4-2026-Q1.csv` — v003 rules, six-month window, default TF

## Quick comparison commands

```bash
# trade count per file
wc -l runs/*.csv

# WR by sweep group across multiple runs (one-liner)
python -c "
import pandas as pd, glob
for path in sorted(glob.glob('runs/v002_*.csv')):
    df = pd.read_csv(path)
    print(f'\n=== {path} ===')
    df['grp'] = df['swept_level_name'].astype(str).str.replace(r'_\d{8}_\d{4}\$','',regex=True)
    print(df.groupby('grp').agg(n=('outcome','size'), wins=('outcome', lambda s:(s=='win').sum()), tot_R=('r_multiple','sum')).to_string())
"
```

## Trade row schema

| Column | Meaning |
|---|---|
| `session_date` | NY trading date |
| `direction` | `short` or `long` |
| `swept_level_name` | Specific sweep target (e.g. `asia_high`, `htf_fvg_1H_bearish_20260311_0300`) |
| `swept_level_price` | The price level itself |
| `sweep_ts` | NY timestamp of first-touch |
| `target_gap_high` / `target_gap_low` / `target_gap_size` | Inversion target FVG |
| `target_gap_formed_at` | When the inversion target FVG formed |
| `bars_to_inversion` | Bars between sweep and trigger (in exec-TF bars, **not** minutes) |
| `entry_ts` / `entry_price` | Inversion-close fill |
| `stop_price` / `tp_price` / `risk_points` | 1R sizing |
| `exit_ts` / `exit_price` / `outcome` | `win` / `loss` / `eod_close` |
| `pnl_points` / `r_multiple` | Per-trade result |
| `exec_tf` | (multi-TF runs only) execution timeframe |
