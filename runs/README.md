# Backtest run outputs

CSVs are **derived artifacts** generated by `scripts/run_backtest.py` (single-TF) or `scripts/run_multi_tf.py` (multi-TF). Tracked in git so trade sets can be diffed across rule versions.

## Folder layout

```
runs/
├── vNNN/
│   ├── <date_range>.csv               ← canonical / "released" run for this version
│   ├── <date_range>_multi_tf.csv      ← multi-TF variant (if applicable)
│   ├── <date_range>_forensic.txt      ← per-day text trace (if generated)
│   └── iterations/                    ← in-development history (chronological)
│       └── <date_range>_NN_<short_label>.csv
└── README.md
```

- **`vNNN/<date_range>.csv`** — the file you compare against when measuring whether v(NNN+1) improved things.
- **`vNNN/iterations/`** — bug-fixes and calibration steps captured during development. Useful for tracing *why* a value changed between releases. Not the file you'd cite as "v002 results."

## Naming convention going forward

```
runs/<rules_version>/<date_range>[_<variant>].csv     ← canonical or named variant
runs/<rules_version>/iterations/<date_range>_NN_<label>.csv  ← in-development
```

`<date_range>` is `YYYY-MM` (one month) or `YYYY-MM-DD_YYYY-MM-DD` (explicit range).
`<variant>` examples: `multi_tf`, `1m`, `30s`, `q1`, `news_filtered`.
`<label>` describes what changed at that iteration step.

## Quick comparison commands

```bash
# Compare canonical v001 vs v002 by sweep group
python -c "
import pandas as pd
for path in ['runs/v001/2026-03.csv', 'runs/v002/2026-03_multi_tf.csv']:
    df = pd.read_csv(path)
    df['grp'] = df['swept_level_name'].astype(str).str.replace(r'_\d{8}_\d{4}\$','',regex=True)
    print(f'\\n=== {path} ===')
    print(df.groupby('grp').agg(n=('outcome','size'), wins=('outcome', lambda s:(s==\"win\").sum()), tot_R=('r_multiple','sum')).to_string())
"

# Show every iteration's trade count side-by-side
wc -l runs/*/iterations/*.csv runs/*/[!iterations]*.csv 2>/dev/null
```

## Trade row schema

| Column | Meaning |
|---|---|
| `session_date` | NY trading date |
| `direction` | `short` or `long` |
| `swept_level_name` | Specific sweep target (e.g. `asia_high`, `htf_fvg_1H_bearish_20260311_0300`) |
| `swept_level_price` | The price level itself |
| `sweep_ts` | NY timestamp of first-touch |
| `target_gap_high` / `target_gap_low` / `target_gap_size` | Inversion target FVG |
| `target_gap_formed_at` | When the inversion target FVG formed |
| `bars_to_inversion` | Bars between sweep and trigger (in exec-TF bars, **not** minutes) |
| `entry_ts` / `entry_price` | Inversion-close fill |
| `stop_price` / `tp_price` / `risk_points` | 1R sizing |
| `exit_ts` / `exit_price` / `outcome` | `win` / `loss` / `eod_close` |
| `pnl_points` / `r_multiple` | Per-trade result |
| `exec_tf` | (multi-TF runs only) execution timeframe |
